Method Overloading

When a class has multiple methods have same name but differ in their number of parameters,their type and format it is known as Method Overriding.Method Overloading increses the readability of program.

By changing no. of arguments
class A{
int mul(int a,int b){
return a*b;
}
int mul(int a,int b,int c){
return a*b*c;
}
}
class Main{
public static void main(String[] args){
A a = new A();
System.out.println(a.mul(2,2));
System.out.println(a.mul(1,2,3));
}
}
Output:
4
6

By changing data types of arguments
class A{
int mul(int a,int b){
return a*b;
}
int mul(float a,float b){
return a*b;
}
}
class Main{
public static void main(String[] args){
A a = new A();
System.out.println(a.mul(2,2));
System.out.println(a.mul(2.5,2.5));
}
}
Output:
4
6.25

Method Overriding

If subclass has same method exactly that of the parentclass then this condition is known as Method Overriding.It is used to implement method which is already defined in parent class and it is use for the runtime Polymorphism.But for method overriding the method must have exactly same name and same parameters and their must be inheritence.

class A{
void m1(){
System.out.println("This is m1");
}
}
class B extends A{
void m1(){
System.out.println("Again this is m1");
}
public static void main(String[] args){
B b = new B();
b.m1();
}
}
Output:
Again this is m1
